40 · 백로그 / 로드맵 — 완료 백로그(v0.2.0) + 향후 로드맵 (ntop)

이 문서는 "무엇을 만들 것인가"가 아니라 "무엇이 이미 만들어졌는가 + 무엇을 더 할 수 있는가"의 단일 출처(single source)다. ntop은 이미 v0.2.0까지 구현·출시된 MIT 오픈소스 도구다(codebase-facts §0). 따라서 백로그는 신규 기획이 아니라 ① 구현 완료된 것(완료 백로그) 과 ② 향후 로드맵으로 나뉜다.

★ 향후 로드맵의 근거는 둘뿐이다(창작 금지): ① codebase-facts §12(코드에 흔적은 있으나 규칙 미등록이거나 한계인 것) ② 선행 문서가 정직하게 기록한 코드 실상 발견(30 §2-12 health 인자 버그, 21 §3-6 version 항상 -, 33/32의 CLI 미마스킹·종료코드 한계 등). 이 둘에 없는 기능은 로드맵에 적지 않는다. 모든 로드맵 항목은 근거 출처 + [가설]/[제안] 라벨을 단다.

참조(재정의 0): 기능 ID(F1~F21)·우선순위(P0/P1/P2)·MVP 경계는 10-prd(§2·§3)와 01-service-plan(§5)이 단일 출처다. Config 기본값(graceful_timeout=10 등)은 31-erd, 알고리즘은 30-functional-spec, 안전·프라이버시 정책은 33-policy, 외부 계약은 32-api-spec, OSS 지속가능성은 00-business-model이 단일 출처다. 본 문서는 그 위에 "완료/예정 상태" 만 얹는다(값·기능 재정의 0).

용어 풀이(첫 등장): 백로그(backlog) = 할 일·구현 항목을 모아 우선순위로 줄 세운 목록 / 로드맵(roadmap) = 앞으로 어떤 순서로 무엇을 할지 그린 지도 / Epic = 여러 기능을 묶은 큰 작업 덩어리 / 간트(Gantt) = 작업을 시간축 막대로 그린 일정표 / MVP(Minimum Viable Product) = 가치를 전달하는 최소 제품 범위 / [가설] = 외부 채택·목표치 등 사실이 아닌 추정 / [제안] = 코드 근거(§12·발견) 위에서 "이렇게 하면 된다"는 개선 제안(아직 미구현) / §12 = codebase-facts의 "미구현/흔적" 절.

3-2. 로드맵 항목 표 (R1~R10) #

우선순위 = 가치(통증 해소 폭) × 근거 강도 × 노력 역수로 판정. 의존은 §5 그래프와 일치. 라벨: [제안] = 코드 근거 위 개선 제안(아직 미구현) · [가설] = 외부 효과/목표 추정.

아래 히트맵은 10개 로드맵 항목(R1~R10)과 2개 정합 항목(M1·M2)을 근거 출처(§12 흔적 / 정직 발견 / 문서·코드 정합)별로 묶고 우선순위(색)로 칠한 것이다. ★ 표시는 팀이 특히 주목한 정직 발견(health 버그·version 미채움·죽은 코드·NAME 배지)이다.
