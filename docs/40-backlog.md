# 40 · 백로그 / 로드맵 — 완료 백로그(v0.2.0) + 향후 로드맵 (ntop)

> **이 문서는 "무엇을 만들 것인가"가 아니라 "무엇이 이미 만들어졌는가 + 무엇을 더 할 수 있는가"의
> 단일 출처(single source)다.** ntop은 **이미 v0.2.0까지 구현·출시된 MIT 오픈소스 도구**다(codebase-facts
> §0). 따라서 백로그는 신규 기획이 아니라 **① 구현 완료된 것(완료 백로그)** 과 **② 향후 로드맵**으로 나뉜다.
>
> **★ 향후 로드맵의 근거는 둘뿐이다(창작 금지):**
> ① **codebase-facts §12**(코드에 흔적은 있으나 규칙 미등록이거나 한계인 것) ②
> **선행 문서가 정직하게 기록한 코드 실상 발견**(30 §2-12 health 인자 버그, 21 §3-6 version 항상 `-`,
> 33/32의 CLI 미마스킹·종료코드 한계 등). **이 둘에 없는 기능은 로드맵에 적지 않는다.** 모든 로드맵 항목은
> **근거 출처 + `[가설]`/`[제안]` 라벨**을 단다.
>
> **참조(재정의 0):** 기능 ID(F1~F21)·우선순위(P0/P1/P2)·MVP 경계는 **`10-prd`**(§2·§3)와 **`01-service-plan`**
> (§5)이 단일 출처다. Config 기본값(`graceful_timeout=10` 등)은 **`31-erd`**, 알고리즘은 **`30-functional-spec`**,
> 안전·프라이버시 정책은 **`33-policy`**, 외부 계약은 **`32-api-spec`**, OSS 지속가능성은 **`00-business-model`**
> 이 단일 출처다. 본 문서는 그 위에 **"완료/예정 상태"** 만 얹는다(값·기능 재정의 0).
>
> **용어 풀이(첫 등장):** **백로그(backlog)** = 할 일·구현 항목을 모아 우선순위로 줄 세운 목록 / **로드맵
> (roadmap)** = 앞으로 어떤 순서로 무엇을 할지 그린 지도 / **Epic** = 여러 기능을 묶은 큰 작업 덩어리 /
> **간트(Gantt)** = 작업을 시간축 막대로 그린 일정표 / **MVP**(Minimum Viable Product) = 가치를 전달하는
> 최소 제품 범위 / **`[가설]`** = 외부 채택·목표치 등 사실이 아닌 추정 / **`[제안]`** = 코드 근거(§12·발견)
> 위에서 "이렇게 하면 된다"는 개선 제안(아직 미구현) / **§12** = codebase-facts의 "미구현/흔적" 절.

## 0. 전제 — "이미 다 만들어졌다"가 출발점

```
 ┌────────────────────────────────────────────────────────────────────────────┐
 │  ntop은 v0.2.0까지 구현·출시 완료된 도구다 (codebase-facts §0)              │
 │                                                                            │
 │   완료 백로그  ███████████████████████████  v0.2.0 = MVP (F1~F21 전부 구현) │
 │   향후 로드맵  ░░░░░░░░░░░ (예정·가설/제안 — §12 + 정직 발견에 한함)         │
 │                                                                            │
 │   "앞으로 만들 것"이 아니라 "무엇이 왜 이렇게 만들어졌나 + 무엇을 더 할 수  │
 │    있나"를 기록한다. 로드맵은 코드 흔적(§12)과 정직 발견 위에서만 자란다.   │
 └────────────────────────────────────────────────────────────────────────────┘
```

- **MVP = v0.2.0 출시 범위 = F1~F21 전부.** MVP 포함/제외 경계의 단일 출처는 `01 §5`·`10-prd §3`다(여기서
  재정의 0).
- **로드맵은 작아야 정직하다.** ntop의 미구현 흔적은 §12에 5건뿐이고, 선행 문서가 코드를 읽다 발견한
  "코드 실상 개선거리"가 5건이다. 합쳐 **10개 기능/수정 로드맵 항목(R1~R10)** 이 전부이며, 그 밖의 항목은
  **근거가 없어 적지 않는다**(spec §7 창작 금지). (별도로 최종 통합 비평이 짚은 **문서·코드 정합 항목
  2건(M1·M2)** 은 §3-3에 분리 기록 — 둘 다 코드 실상 근거가 있다.)

## 3. 향후 로드맵 — §12 흔적 + 정직 발견만 (R1~R10)

> (코드에 흔적 있으나 미등록/한계)에서, R6~R10은 **선행 문서가 코드를 읽다 정직하게 기록한 코드 실상**
> (30/21/32/33)에서만 도출했다. 각 항목은 **근거 출처 + 라벨(`[가설]`/`[제안]`)** 을 단다.

- **단일 출처 준수:** Config 값(`graceful_timeout=10`·`MAX_BUFFER_LINES=1000`)은 값으로 적지 않고 31 인용,
